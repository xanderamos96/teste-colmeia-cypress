Neste projeto foi criado varias automações que validam as funcionalidades da plataforma disponibilizada.
Foi criado arquivos gherkin para roteirizar o funcionamento de cada feature para uma melhor validação.

Os testes foram separados em 3 categorias:
Success: Testes aonde o sistema se comporta em cenários em que a feature se comportou da forma correta (fluxo feliz)
Fail: Teste aonde o sistema se comporta de forma correta mas por algum erro ou manuseio incorreto do usuário. Usuário deixar de preencher algum campo e querer salvar algum item por exemplo, o sistema irá dar uma mensagem de erro.
Bug: Teste aonde o sistema não esta funcionando de forma correta. Sendo assim considerado um bug/erro
